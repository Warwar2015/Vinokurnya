## 📋 Ручное тестирование

### F-01: Запуск приложения
**Инструкция:**
```bash
cd Vinokurnya
dotnet run
```

**Ожидаемый результат:**
- Приложение открывается без ошибок
- Окно "Винокурня" загружается
- Видно главное окно с заголовком и вкладками

**Проверить:**
- [ ] Нет ошибок компиляции
- [ ] Приложение стартует
- [ ] Появляется главное окно
- [ ] Загружены данные (рекомендуемые рецепты)

---

### F-02: Создание рецепта
**Инструкция:**
1. Перейти на вкладку "📖 Рецепты"
2. Нажать кнопку "Добавить рецепт" (если есть)
3. Заполнить форму:
   - Название: "Тестовый рецепт"
   - Категория: "Виски"
   - Крепость: 60%
4. Нажать "Сохранить"

**Ожидаемый результат:**
- Рецепт появляется в списке
- Показывается сообщение "Успешно"
- Рецепт сохранен в базе данных

**Проверить:**
- [ ] Форма валидирует обязательные поля
- [ ] Рецепт отображается в списке
- [ ] Рецепт сохранен и виден после перезапуска
- [ ] Иконка избранного работает

---

### F-04: Редактирование рецепта

3. Нажать "Сохранить"

**Ожидаемый результат:**
- Изменения сохраняются
- Рецепт в списке обновляется

**Проверить:**
- [ ] Данные редактируются
- [ ] Изменения сохраняются
- [ ] Обновляется UpdatedAt

### F-07: Импорт JSON
**Инструкция:**
1. Создать файл `test_recipes.json` с валидными данными:
```json
[
  {
    "name": "Тестовый рецепт 1",
    "category": "Виски",
    "difficulty": 3
  }
]
```
2. Открыть приложение → Рецепты → Импорт

**Ожидаемый результат:**
- Данные корректно добавляются
- Рецепты появляются в списке

**Проверить:**
- [ ] Файл импортируется
- [ ] Рецепты добавляются
- [ ] Даты создания правильные
